# La Buena Mesa — web estática

Este proyecto está preparado para publicarse en GitHub Pages.

## Archivos principales

- `index.html`: estructura de la página.
- `styles.css`: estilos visuales, responsive y estética de marca.
- `script.js`: menú móvil y header con fondo al hacer scroll.
- `assets/`: carpeta para imágenes y logo.

## Imágenes que debes reemplazar

Dentro de `assets/`, añade tus imágenes con estos nombres exactos:

- `logo.png`
- `hero.jpg`
- `hero-mobile.jpg`
- `gallery-1.jpg`
- `gallery-2.jpg`
- `gallery-main.jpg`
- `gallery-3.jpg`
- `gallery-4.jpg`
- `landscape.jpg`

Incluí `diseno-referencia.png` solo como referencia visual. No es necesario publicarla.

## Cómo publicar en GitHub Pages

1. Crea un repositorio nuevo en GitHub, por ejemplo `la-buena-mesa`.
2. Sube `index.html`, `styles.css`, `script.js` y la carpeta `assets`.
3. Entra en `Settings` → `Pages`.
4. En `Build and deployment`, elige:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/root`
5. Guarda.
6. Espera 1–3 minutos y abre la URL que te da GitHub Pages.

## Consejos de imagen

Para que cargue rápido:

- Hero: 1800px de ancho aprox.
- Galería: 900–1200px de ancho aprox.
- Peso ideal por imagen: menos de 400 KB.
- Formato recomendado: `.jpg` para fotos, `.png` para logo con transparencia.
